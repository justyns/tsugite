# v0.17

## 0.17.0 (2026-06-29)

### Features

- tsugite is now a light kernel plus swappable plugins. `pip install tsugite-cli` gives you the CLI, built-in tools, and providers; the `[daemon]`, `[web]`, `[sandbox]`, `[discord]`, and `[all]` extras add the rest. The daemon, web search, YouTube transcripts, the bubblewrap sandbox, terminals (PTY), the Claude Code provider, and Discord each live in their own `tsugite-*` package.
- Subsystems are pluggable behind a uniform seam (a protocol, a `tsugite.<area>` entry point, and a `config.<area>.backend` selector). History, attachments, context injection, sandbox, and executor join the existing tool/provider/adapter/secret/hook batteries, so new backends can ship as plugins.
- Conversation history is stored in one SQLite database instead of a JSONL file per session, searchable with `tsu history search` and shared across run, chat, workspace, and daemon mode. A turn no longer re-stores the whole conversation, so the data is smaller.
- Branch a conversation: fork it at a chosen point into an independent continuation without changing the original (daemon API).
- `tsu history export <id>` writes a conversation back to JSONL on demand.
- Agents tolerate uninstalled tools: a tool from a plugin you don't have is skipped with a warning instead of failing the run. Set `strict_tools: true` in frontmatter to require every listed tool.
- `tsu validate` reports tools an agent declares but that aren't installed (a warning, or an error under `strict_tools`).
- `sandbox.pass_env` exposes specific environment variable names to a sandboxed agent on top of the safe defaults.
- The package ships a `py.typed` marker so plugin authors' type checkers pick up tsugite's inline types.

### Breaking changes

- `pip install tsugite-cli` no longer includes the daemon, web search, or sandbox. Install `tsugite-cli[daemon]` (or `[all]`) to restore the previous full setup.
- The `--ui live` output mode was removed. The default plain mode and `--headless` remain.
- SQLite is the default history backend and `jsonl` is deprecated. Run `tsu history import` once to copy existing JSONL sessions into the database; the original `.jsonl` files are kept as a backup.
- Workspace sessions now live in the shared history database, not a per-workspace `session.jsonl`.
- Keep tsugite's data directory on local disk; SQLite history can corrupt on a network filesystem (NFS/SMB).

### Notable fixes

- Sandboxed agents no longer inherit the operator's environment: bwrap clears it and re-adds only a safe allowlist, so API keys and tokens in the environment can't be read by agent code. Secrets stay available through the masked `get_secret` tool.
- The tmux tools and the schedule remove/disable/cleanup tools are denied to sandboxed agents, and PTY sessions and spawned sub-agents/jobs inherit the parent's sandbox.
- Positional tool-call arguments work consistently across executors; passing both a positional and a keyword for the same argument now raises instead of silently overwriting.
- The web UI's Stop button now actually stops the agent: the run loop checks for a cancel between turns and before each tool/code step, so a stopped turn no longer keeps running (and writing history) in the background. A Stop mid-generation still finishes the current model call, then stops.
- The carried-forward conversation summary is one height-capped card that scrolls internally instead of pushing the conversation off-screen, and it no longer vanishes (along with the "load more" pagination button) when you re-open a session from the sidebar.
- The reasoning-effort selector re-derives its options when you change a session's model, so it shows the selected model's actual effort levels instead of the agent default's.
- `ask_user` / `ask_question` prompt buttons are styled to match the rest of the web UI instead of rendering as bare browser buttons.
- Stuck or errored Jobs can be dismissed (cancelled) from their tile, instead of being stranded with only "retry" or a misleading "mark done".
- A Job whose verifier returned its verdict JSON twice back-to-back is no longer wrongly marked stuck; the first verdict is used.
- Scheduled-run history cleanup no longer deletes the live conversation of an active schedule that reuses a fixed session id.
- `tsu history migrate` merges legacy daemon events (reactions, prompt snapshots) in timestamp order instead of appending them after the conversation, so migrated sessions read chronologically.
