# 0.13.0

The web UI got a rewrite, MCP support is gone, and a bunch of session/compaction/skills/etc improvements and bugfixes.

## Web UI

The daemon UI was rewritten from scratch, mostly from an experiment with Claude Design. Highlights:

- Live session progress in the sidebar; status text, pulse, and topic show inline.
- Reasoning and per-turn thought render as first-class bubbles, instead of bundling in a single tool expander.
- Session pinning and renaming.
- Per-session topic (160 char) that the model or user can update.
- Send messages to multiple sessions in parallel.
- Better compaction progress and link to the new session if needed.
- Mobile improvements.
- Model dropdown with filtering/scrolling.
- Full `tool_result` output in history (no more 500-char truncation), XML entity decoding.

## Other features

- Unified `reasoning_effort` across providers, exposed in the UI.
- Multistep agent files now take `agent=` on a step to delegate it to another agent via `spawn_agent`.
- `tsu skill check` reports validation issues from the CLI.
- Failed skills show as a badge in the UI and as a `<failed_skills>` block in the agent's context.
- Trigger-based skills auto-load via regex matching.
- `attachments` frontmatter gained `assign:` and `mode: index`.
- Prefetch `args:` render through Jinja; `get_secret` is exposed to templates.
- `run` tool takes an `env=` argument.
- `httpresponse.json()` method; `.text` no longer auto-parses JSON.
- `gpt-5.5` and `opus-4.7` added to the model list.
- "Waiting on LLM (Ns)" heartbeat while waiting on slow llms.
- Per-event JSONL session log replaces the old Turn aggregate (`tsu history migrate` converts older files).  This does increase the log file size, but makes things a lot easier.

## Removed

- MCP support, both directions. `tsu serve mcp`, the `mcp_servers:` frontmatter field, the `mcp` and `serve` CLI subcommands, and `--trust-mcp-code` are all gone. Might be re-added as a plugin later.

## Notable fixes

- Parser handles multiple code blocks better. Uses `ast.parse` to find the real closing fence instead of relying on regex.
- Fix duplicate `http_request` POSTs and `write_file` calls caused by some code being re-run twice.
- Dropped extra code blocks from multi-block responses and let the llm know.
- Improve default system prompt and default agent
- Compaction now isolates provider state so attachments don't leak between summarization and the next turn. The post-compacted session has an estimated token count now too and should be correct after the first message.
- Several fixes to allow running multiple sessions concurrently
- `http_request` defaults to `GET` instead of silently returning empty data on POST-shaped calls without a body.
- Rejected LLM responses now show in the UI.
