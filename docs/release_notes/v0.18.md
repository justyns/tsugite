# v0.18

## (unreleased)

### Features

- Daemon adapter plugins can register HTTP routes (under `/api/plugins/<name>`) and non-agent **job executors**. A job with a registered `executor=` reuses the full job machinery: verification, retry, stuck/mark-done/cancel, and the web tile. See `docs/plugin-adapters.md`.
- New **`tsugite-cc-driver`** plugin: run an interactive Claude Code session as a verified job (`executor="cc"`). The job tile embeds the live terminal, so you can watch it and take over by typing. Optional bubblewrap sandboxing. See `plugins/tsugite-cc-driver/README.md`.
- Jobs can pause for input: a blocked worker parks in `awaiting_input` and wakes its parent session with the question. `respond_to_job()` answers and resumes it; `cancel_job()` abandons it. The sidebar shows a persistent "needs your input" indicator, and an unanswered question still times out to `stuck`.
- Per-job reasoning `effort` (`/job ... --effort max`, or `spawn_job(effort="max")`); the cc executor passes it to claude's `--effort`.
- Jobs board tiles have per-tile cancel/retry actions, so you can act on a job without opening its session.
- The OpenAI model registry knows the `gpt-5.6` family (Sol / Terra / Luna tiers plus the bare `gpt-5.6` alias that routes to Sol), including the new `max` reasoning-effort level, sourced from models.dev.

### Fixes

- A conversation no longer wedges permanently when a resumed Claude Code transcript is rejected with `400 text content blocks must be non-empty`: the provider falls back to a fresh session seeded from tsugite's serialized history, and empty messages are never sent to the CLI (so tsugite can't poison a transcript in the first place).
- Scheduled and subprocess sessions that end via `return_value(...)` now show the returned value in the conversation view, not just the sidebar - the final answer is recorded durably in the history event log on every run path.
- Waking the mobile PWA no longer flashes the conversation several times: a burst of reconnect events coalesces into one non-destructive catch-up that refreshes the thread in place instead of blanking and rebuilding it per event.
- After a mobile PWA cold restart, the app reopens the session you last had open: the persisted selection resolves by conversation id like the URL-hash path, and the auto-select fallback no longer overwrites your last-viewed preference with the pinned pick.
- Compaction preserves user-authored `task`/`pr`/`notes` session metadata on the successor session (transient `status_text` still resets).
- Removed dead UI affordances: the ⌘K "command palette" button and keystrip hint (it only toasted "coming soon"), the phantom `@ agent` / `# file` composer hints, and the wrong `⌘↩ send` shortcut label.

- `tsu:exec` code blocks run under a wall-clock timeout (default 300s, `TSUGITE_EXEC_TIMEOUT`), so a hung tool call fails the block instead of hanging the turn.
- Mounted API endpoints (`/api/jobs`, `/api/terminals`, etc.) work behind a reverse proxy again (no more absolute internal-host redirects).
- The sidebar dot pulses only while a turn is running, not for every active session.
