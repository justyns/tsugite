# v0.15

## 0.15.0 (unreleased)

Jobs and a PTY terminal viewer are the headline features. A lot of fixes besides.

### Jobs (new feature)

Chat-spawned background work with a verification loop. `/job <prompt> [--ac "tests pass"]` (or `spawn_job()` from an agent) spawns a worker; on completion a reasoning-blind verifier grades it against the acceptance criteria and marks it `done`, retries, or `stuck`. Jobs render as inline tiles in the parent chat and on a dedicated Jobs tab.

- State machine `queued → running → verifying → done | stuck` (plus `cancelled` / `errored`), persisted to `<state_dir>/jobs.json`.
- Acceptance criteria can be predicates (`cmd:`, `exit_code:`, `file_exists:`) or prose graded by the verifier.
- `--repo <path>` runs the worker and verifier in a fresh `git worktree`, isolated from the parent checkout.
- Tile actions: cancel, retry-with-hint, mark-done. Orphaned jobs from a previous daemon are marked `errored` on boot.
- Agent tools `spawn_job` / `get_job` / `list_jobs`; HTTP `POST /api/jobs/<id>/{cancel,mark-done,retry}`.

### Terminal viewer (new feature)

`/run <cmd>` (or the `pty_create` / `pty_send_keys` / `pty_capture` / `pty_kill` agent tools) spawns an interactive PTY shell that streams live via SSE into a sidebar viewer with kill / restart / follow.

### Providers and models

- `tsugite-codex-cli` plugin: use a ChatGPT Plus/Pro subscription as a provider (`codex_cli:gpt-5.5` and friends). One-time `codex login`; see the plugin README.
- `claude-opus-4-8` added to the Claude Code provider.

### Agent config

- New `model_kwargs` frontmatter field for provider-passthrough kwargs (e.g. `response_format`), merged across inheritance.
- User global agents (`~/.config/tsugite/agents/`) now win over plugin-shipped agents of the same name.

### Notable fixes

- Secrets from `get_secret()` are masked in the persisted event log, not just the live view.
- Model-fabricated runtime tags (`<tsugite_execution_result>`, etc.) are escaped before storage, so they no longer pollute context, compaction, or the UI.
- Claude Code subprocess: 16 MB stream-read limit so long output lines no longer crash it; `stop()` escalates to SIGKILL so wedged workers don't leak.
- `state = {...}` rebinds in agent code persist again under `LocalExecutor`; a `<content>` block can no longer shadow a tool of the same name.
- Scheduler rejects a bad timezone / cron / `expires_at` at create instead of bricking later; missed one-offs and corrupt entries are reaped.
- Background-task results are delivered on webhook and web-push channels, not just Discord.
- Auto-compaction skips sessions mid-turn and won't re-compact an already-superseded session.
- Web push: VAPID claims are copied per send, so delivery no longer breaks after the first provider.
- Webhooks: `source` is validated, payloads capped at 5 MB, and writes happen off the event loop.
- Uploads are rejected by Content-Length before buffering and survive the auto-compact retry.
- Service worker no longer force-reloads on first activation (it was killing in-flight chat streams).
- A reload mid-turn no longer leaves a permanently-spinning "running" bubble.
- `/status`, `/context`, and `/compact` act on the open chat, not just the primary session.
- Jobs tab scrolls when it overflows; a tile no longer sticks on "running" after opening the worker.
- `list_agents` no longer reports stray frontmattered `.md` files in the workspace root as spawnable agents.
- Conversation breadcrumb session-id segment is click-to-copy.
