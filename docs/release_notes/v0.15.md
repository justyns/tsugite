# v0.15

## 0.15.0 (unreleased)

### Jobs (new feature)

Chat-spawned background work with a verification loop. Type `/job <prompt> [--ac "tests pass|PR open"]` in the web UI (or call `spawn_job()` from an agent) to spawn a worker session that runs in the background. On completion, a reasoning-blind verifier sub-agent judges the result against the acceptance criteria and either marks the Job `done`, retries the worker (up to 3 attempts), or marks it `stuck` with an AC breakdown.

- Six-state machine: `queued → running → verifying → done | running (retry) | stuck`, plus `cancelled` / `errored` escape hatches. Persistent JSON-backed `JobStore` at `<state_dir>/jobs.json`; legacy `state: looping` from pre-MVP daemons is migrated to `running` on load.
- `JobsOrchestrator` subscribes to `SessionRunner.notify_callback`, drives state transitions, spawns the verifier, and broadcasts tile updates.
- Built-in agents bundled with the package: `job_worker` (structured-summary discipline) and `job_verifier` (reasoning-blind, strict JSON output via `response_format: json_object`, hermetic via `extends: none`).
- Inline `job_status` tile in the parent chat: live SSE updates keyed by `job_id`, persisted into the parent session's JSONL so the tile re-renders on reload. State-coloured borders (lavender → green → red).
- Wall-clock timeout per job (default 30 min, configurable per-call) covering both the worker and the verifier window. Worker spawns route through the parent session's adapter (no cross-agent credential/tool leakage). Worker output is preserved even on user-cancellation / external state changes.
- **`--repo` enforcement**: when a Job is spawned with `--repo <path>`, the orchestrator provisions a fresh `git worktree add --detach` at `<repo>/.tsugite-jobs/<job_id>` and binds the worker session's workspace to that directory. Concurrent jobs in the same repo no longer trip over each other. Worktrees are pruned on DONE/CANCELLED; kept on STUCK/ERRORED for inspection.
- **Tile actions**: `running`/`verifying` tiles get a `cancel` button (cancels worker + verifier, transitions Job to CANCELLED). `stuck` tiles get `mark done` (override the verifier with an audit reason) and `retry…` (hint-and-retry — provide a one-shot hint, do NOT reset `verify_attempts`).
- New HTTP endpoints: `POST /api/jobs/<id>/cancel`, `POST /api/jobs/<id>/mark-done`, `POST /api/jobs/<id>/retry`.
- **Parent-agent awareness**: every user message in a chat now includes a `<jobs>` block in `<message_context>` listing the chat's active jobs and last 3 terminal jobs (id, state, prompt[:80], timestamps, worker/verifier session ids, error if stuck). The LLM can call new `get_job(job_id)` / `list_jobs(...)` tools for full details. No worker output is dumped into the chat; the tile + tools are the surfaces.
- **Push notification on completion**: optional one-line wake-up message posted into the parent session when a Job hits a terminal state ("Job xxx finished with state 'done': <prompt>. Use get_job('xxx') for details."). Default is OFF for `/job` (use `--notify` to enable), ON for `spawn_job()` agent tool (so the parent agent can react to its own background work).
- New `Session.workspace_override` field: per-session working-directory override, threaded through the adapter so the worker's PathContext / fs tools / shell tools all land in the override directory.

### New providers and models

- `tsugite-codex-cli` plugin: use a ChatGPT Plus/Pro subscription as a tsugite model provider, routing through the Codex Responses API. Requires a one-time `codex login` to populate `~/.codex/auth.json`; the plugin reads and refreshes that file in place under a sidecar file lock. Models: `codex_cli:gpt-5.5`, `codex_cli:gpt-5.4`, `codex_cli:gpt-5.4-mini`, `codex_cli:gpt-5.4-nano`. Streaming, reasoning content, and per-turn token accounting all wire through. See `plugins/tsugite-codex-cli/README.md` for the token-sink caveat and recovery instructions.
- `codex_cli:gpt-5.5` added (1.05M input tokens, 128K output, vision, reasoning with `low | medium | high | xhigh` effort levels).

### Agent config

- New `AgentConfig.model_kwargs` field (`Dict[str, Any]`): provider-passthrough kwargs declared once in an agent's frontmatter (e.g. `model_kwargs: {response_format: {type: json_object}}`) and threaded into every invocation. Shallow-merges across inheritance. Precedence for `reasoning_effort` clarified: CLI/exec override > caller kwargs > agent.model_kwargs > agent.reasoning_effort field.

### Web UI

- Conversation breadcrumb session-id segment is now a clickable button — click copies the full session ID to the clipboard. Hover shows the full ID via the `title` attribute (#320).

### Daemon

- `SessionRunner._run_session`'s CancelledError branch now invokes `notify_callback` (symmetric with the COMPLETED / FAILED branches). External `cancel_session` calls now reach subscribers (including the Jobs orchestrator) instead of silently terminating.
