## 0.15.0 (unreleased)

### Jobs (new feature)

Chat-spawned background work with a verification loop. Type `/job <prompt> [--ac "tests pass|PR open"]` in the web UI (or call `spawn_job()` from an agent) to spawn a worker session that runs in the background. On completion, a reasoning-blind verifier sub-agent judges the result against the acceptance criteria and either marks the Job `done`, retries the worker (up to 3 attempts), or marks it `stuck` with an AC breakdown.

- Six-state machine: `queued â†’ running â†’ verifying â†’ done | running (retry) | stuck`, plus `cancelled` / `errored` escape hatches. Persistent JSON-backed `JobStore` at `<state_dir>/jobs.json`; legacy `state: looping` from pre-MVP daemons is migrated to `running` on load.
- `JobsOrchestrator` subscribes to `SessionRunner.notify_callback`, drives state transitions, spawns the verifier, and broadcasts tile updates.
- Built-in agents bundled with the package: `job_worker` (structured-summary discipline) and `job_verifier` (reasoning-blind, strict JSON output via `response_format: json_object`, hermetic via `extends: none`).
- Inline `job_status` tile in the parent chat: live SSE updates keyed by `job_id`, persisted into the parent session's JSONL so the tile re-renders on reload. State-coloured borders (lavender â†’ green â†’ red).
- Wall-clock timeout per job (default 30 min, configurable per-call) covering both the worker and the verifier window. Worker spawns route through the parent session's adapter (no cross-agent credential/tool leakage). Worker output is preserved even on user-cancellation / external state changes.

### New providers and models

- `tsugite-codex-cli` plugin: use a ChatGPT Plus/Pro subscription as a tsugite model provider, routing through the Codex Responses API. Requires a one-time `codex login` to populate `~/.codex/auth.json`; the plugin reads and refreshes that file in place under a sidecar file lock. Models: `codex_cli:gpt-5.5`, `codex_cli:gpt-5.4`, `codex_cli:gpt-5.4-mini`, `codex_cli:gpt-5.4-nano`. Streaming, reasoning content, and per-turn token accounting all wire through. See `plugins/tsugite-codex-cli/README.md` for the token-sink caveat and recovery instructions.
- `codex_cli:gpt-5.5` added (1.05M input tokens, 128K output, vision, reasoning with `low | medium | high | xhigh` effort levels).

### Agent config

- New `AgentConfig.model_kwargs` field (`Dict[str, Any]`): provider-passthrough kwargs declared once in an agent's frontmatter (e.g. `model_kwargs: {response_format: {type: json_object}}`) and threaded into every invocation. Shallow-merges across inheritance. Precedence for `reasoning_effort` clarified: CLI/exec override > caller kwargs > agent.model_kwargs > agent.reasoning_effort field.
