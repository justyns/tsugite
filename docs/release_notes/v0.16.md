# v0.16

## 0.16.0 (2026-06-20)

### Features

- `tsu exec` runs tsugite skills and agents from an external coding agent (Claude Code, Cursor, Aider) inside tsugite's tool namespace, so skill code that calls `get_secret`, `http_request`, or `run` works as-is.
- Per-agent sandboxing now applies to every daemon execution path. An agent can tighten its own sandbox in frontmatter but never loosen it.
- Doc skills can opt out of Jinja rendering with `jinja: false`.

### Notable fixes

- History replay no longer shows the tail of an executable `python` block as assistant prose.
- Replayed code-execution output is truncated to match the live turn.
- Compaction summary is no longer duplicated in the chat history.
- claude_code provider no longer doubles streamed response text.
- Anthropic cache tokens are counted in usage totals, streaming and non-streaming.
- Streaming usage is captured when it rides the finish chunk's choices.
- Model-registry prefix matching rejects unrelated variants, so an unlisted model can't inherit a sibling's pricing.
- `read_file` honors `end_line` on its own and clamps line metadata to EOF.
- Overlapping schedule fires are suppressed without corrupting drift metadata.
- Repeatedly-failing schedules are surfaced instead of failing silently.
- Compaction repoints the channel index so channel history isn't orphaned.
- Agents get the `uploads/` path for non-inlined uploads.
- Frontmatter sandbox domain caps are port-aware, glob-matched, and inherit terminal policy.
- Web UI: no duplicate "Working" bubble when opening a busy session.
- Web UI: history refetches on tab resume.
- Web UI: follow-tail stays pinned during streaming.
- Web UI: terminal keystrokes reach the PTY stdin endpoint.
- Web UI: browser back/forward switches conversations.
