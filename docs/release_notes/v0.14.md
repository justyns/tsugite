# v0.14

## 0.14.3

### Breaking changes

- `daemon.yaml` no longer accepts plaintext `token:` on `discord_bots[]`. Each bot must now set exactly one of `token_secret: <name>` (resolved via the secrets store) or `token_file: <path>`.

### Notable fixes

- `tsu run` no longer leaks status messages, warnings, and errors to stdout.
- Daemon HTTP API: `GET /api/schedules` no longer fails with `Object of type Lock is not JSON serializable`.
- Web UI sidebar live label surfaces compacting state, so you can see at a glance which sessions are currently being compacted.
- Sidebar unread badge no longer reappears after stream-end when the agent emits housekeeping `session_metadata` calls (e.g. `status_text='idle'`).
- Auto-compaction interprets its cron in the agent's timezone instead of UTC, and also fires on large idle sessions (not just running ones).
- Post-compaction events no longer carry a stale claude_code `session_id` that would `--resume` into a finished subprocess.
- Web UI in-flight streaming state survives auto-compaction by following the supersession alias to the live successor session.
- Web UI shows `preparing to compact` / `summarizing N turns` during the compaction race window instead of going blank.
- `context_limit` is now stored per-session and snapshotted around the full compaction body, so a smaller-model side call can't clobber what other sessions display.
- Changing an agent's `model:` in config now freezes existing sessions to their previous model so an in-flight conversation doesn't break mid-stream.

## 0.14.1

### Features

- New ACP provider plugin under `plugins/tsugite-acp/` for talking to Agent Client Protocol servers.
- Bidirectional compaction-chain pointers: `compacted_into` on the old JSONL, `source_session_id` on the new one. Web UI navigates both directions; "continue in live session" link on compacted threads.
- New `--ui live` mode renders a three-region terminal layout: scrollback log lines flow above a persistent 1-3 line status footer showing `Turn N | tool short_desc | tokens K | $cost | M:SS`. Footer auto-collapses to a single line on short terminals and degrades to plain when stdout is not a TTY.

### Notable changes

- `tsu run` defaults to plain output instead of the rich panel UI; pass `--ui live` for the new persistent status footer. `--plain`, `--headless`, and pipe/NO_COLOR auto-degrade behavior unchanged.
- Web UI scrolls to bottom on session switch instead of landing mid-conversation on revisit.
- Sidebar `Turn N · Thinking…` clears after the turn ends. `loadSessions` reconciles against the server's cleared `status_text` so a missed SSE can't keep the cache alive.
- POST `/chat` follows compacted successors instead of returning HTTP 409. Web UI auto-switches on `session_update` with `action: 'compacted'`.
- Cancelled default sessions no longer auto-respawn on the next inbound request from the same user.
- Retired `_interactive_index` + `get_or_create_interactive` + "Main Session" literal. Default routing is driven entirely by the `is_primary` metadata flag added in 0.14.0; new helpers `find_default_session()` and `create_default_session()` replace the legacy fallback across HTTP, Discord, base adapter, and command callsites. Legacy stores get `is_primary` stamped on the most-recently-active interactive session per (user, agent) at load.
- Jinja `env` is now callable in agent templates with an optional default: `{{ env('FOO', 'fallback') }}`. The dict-access form (`{{ env.FOO }}` / `{{ env['FOO'] }}`) still works.

### Notable fixes

- Web UI restores the user's last-selected session after a PWA cold restart at the manifest's `start_url: "/"` (e.g. iOS evicting the standalone app from memory). Previously reopened on whatever `autoSelectInteractive` ranked first - rarely the conversation you had open. Per-agent `tsugite_selected_session_<agent>` localStorage key; `selectSession`'s existing `superseded_by` chase handles stale persisted IDs for free.
- File uploads in the web UI no longer return 500 - `python-multipart` is now a daemon dependency.
- Untitled sessions in the sidebar fall back to date or prompt instead of repeating the same `Web: web-…` username prefix once per row.
- Push-notifications toggle surfaces a timeout/error in the UI instead of silently hanging when the daemon doesn't respond.
- Mid-turn `history_update` SSE reloads no longer race the per-chat streaming reader's `final_result` push (no more duplicate or vanishing trailing turn).
- Cleared serious/critical axe-core a11y violations across the major console views; new a11y ratchet test in `tests/e2e/test_a11y.py` keeps regressions out.

## 0.14.0

Plugin system overhaul, primary/target session routing for schedules, Discord DM routing per user, faster web-UI session switching, and several fixes.

### Plugins

- Unified entry point: a plugin's `__init__.py` returns a `Plugin` object that the loader wires up. `@tool`, `@hook`, and `@subscribe` decorators replace the older registration style.
- The tmux integration is now a core plugin instead of being built in.  I plan on moving more things to a plugin system to trim the core down.
- Plugin authoring guides under `docs/`.

### Other features

- `tsu:exec` agent directive runs inline Python at render time.
- Time-grounding signals (current time, file mtimes, replay timestamps) flow into message context, file tools, and replay.
- New `history_update` and `session_update` SSE events fire after each adapter turn so live UIs stay in sync without polling.
- Runtime Dockerfile for image publishing.
- Schedules accept a `target_session:` field (resolver included), and `target_session` is plumbed through tools, the CLI, and the HTTP API. Schedules can now drop work into a specific named session instead of always spinning up a fresh one.
- Resizable sidebar. Hover-to-expand removed so rows do not jiggle when the cursor sweeps across them.
- Top sessions prefetch on boot, so the first click usually renders instantly. Added a loading spinner too.
- Per-tool steps render live in the progress bubble (parity with Discord).
- Discord
  - DMs route to a per-user named session (default `discord`). Channels and threads still use the shared session.
  - Progress header shows the session route; compaction states are split; new `unified_routing` setting.
  - Tool names, LLM-waiting heartbeats, and thought prose surface in progress.

### Notable fixes

- Compaction happens more reliability on long sessions.
- Cross-session SSE bus no longer broadcasts turn-end events to other sessions (fixes duplicate live messages in the web UI).
- Tool-call arguments are masked (if they contain secrets) before event-bus emission.
- Web-UI session-switching speedups: progress and event_count cached, history fetch short-circuits on limit, single-pass `collect_events`, progress cache evicts on finish.
- Per-step SSE broadcast for adapter turns; `session_end` clears `status_text`.
- claude-code adapter raises `AgentExecutionError` on CLI `is_error` results instead of silently returning.
- `__version__` now read from package metadata.
- lxml 6.1.0 (CVE-2026-41066).
